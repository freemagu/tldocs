## Decisions made (and why)

- **AUD-0187 policy: never retry ambiguous POST failures pre-AUD-0039.**
  Without `orderLinkId`, a `ReadTimeout` that actually landed on Bybit is
  indistinguishable from a refused request. Retrying risks a duplicate order.
  Conservative: all non-success outcomes → guard FAILED immediately. Removed
  the old 3-strike circuit breaker as dead code.

- **AUD-0184 transaction scope excludes the Bybit cancel call.** Holding
  DB row locks across a network round-trip is a deadlock vector. Cancel runs
  with `autocommit=True`; only the DB cascade (archive → attempt → guard
  update → leg delete) is wrapped.

- **Bundle D extracted `_finalize_successful_execution` as a behavior-
  preserving refactor** to make the cascade testable in isolation — the prior
  inline shape in `_apply_event` was impossible to unit-test without
  replicating the entire event-processing loop.

- **AUD-0196 verified as non-bug.** Git blame (`74412843`, 2026-03-19)
  predates the audit (2026-04-22). Python guarantees `except` runs before
  `finally`, so state mutations in except branches execute inside the lock
  scope. Pinned with a regression test; no production change.

- **Bundle 2a pool sizing: `DEFAULT_POOL_MAX=20`.** Matches the audit's
  "20 guards" scenario; combined with core `pg_pool` (maxconn=10) stays
  well under PG default `max_connections=100`.

- **AUD-0306 left as-is; invariant pinned.** Current 30s/120s ratio
  satisfies `refresh*3 ≤ window`. Test catches any future tuning that
  drops below a 3:1 margin.

- **Smoke test scope: read-heavy on Bybit.** `_cancel_exchange_order` is
  the highest-risk path for AUD-0188; execute path is already covered by
  `bybit_mock` which exercises real `BybitClient._request` at the httpx
  layer. No positions/market orders in the smoke test, so cleanup is a
  single cancel call.

## Surprises / gotchas

- **`BybitClient.place_order` uses snake_case `order_link_id`**, not the
  camelCase `orderLinkId` from Bybit docs. Test code should match the
  Python wrapper's param, not the raw API.
- **`bybit_demo` is in hedge mode** — `position_idx=0` (one-way default)
  fails with `position idx not match position mode`. Must call
  `BybitClient.detect_position_mode(category, symbol)` and pick 0 or 1/2
  accordingly. Production already does this; my smoke test initially
  hardcoded and had to be fixed.
- **Bybit rejects limit prices too far from market** as "price is invalid".
  Use a ticker-derived percentage (e.g., 50%) instead of a hardcoded low.
- **`PostgresDB` default is `autocommit=True`.** Matters for Bundle E's
  transaction wrap — we toggle to False, do the cascade, commit/rollback,
  then restore True.
- **`test_db_conn` (rollback fixture) conflicts with autocommit toggle.**
  Code that calls `conn.autocommit = False` while a tx is already implicitly
  open from the seed INSERT raises `ProgrammingError`. Integration tests
  of tx-wrapping code need the `api_seeded` pattern (separate autocommit
  conn for seed).
- **`bybit_client.py` briefly had a `NameError: name 'logging' is not
  defined`** at line 194 — introduced by an earlier session's AUD-0003
  redaction work that referenced `logging.DEBUG` without importing the
  stdlib module. Another session fixed it in `12541a09` mid-work.
- **`_determine_execution_strategy` ALREADY raises ValueError and the
  caller already handles it** — AUD-0189's "silent removal" description
  slightly overstates; the fix is the log-severity upgrade, not new
  error handling.
- **`acquire_singleton_lock` was already implemented** in
  `lib/tradelens/utils/singleton_lock.py` and used by 4 other daemons
  before Bundle 1. Bundle A was "import and call" in 3 files, not a new
  helper.
- **Daemon `_execute_market_order` inherits `position_idx` from the leg
  row** — doesn't call `detect_position_mode`. That's correct: the
  replacement order must match the original's positionIdx, and the
  original's value was set correctly at creation time by user-facing
  code paths in `api/trades.py` / `api/open_orders.py`.
