## Decisions made (and why)

- **Sub-agent-per-batch execution shape.** Each sub-agent got the plan, the triage, and a scoped set of AUD-IDs. Kept parent context light (I still had plenty of headroom at session end). Groups 1-10 each handled 2-9 batches.
- **3 parallel triage agents for chunks 3-14.** The chunks split roughly by file ownership (3-5 API writers / 6-9 backend subsystems / 10-14 frontend + ops). Parallel classification saved wall-clock; merging was straightforward because each wrote to its own file.
- **Serial execution, not parallel worktrees.** AUDIT_TRACKER.md is the single point of contention — parallel sub-agents would conflict on it. Chose serial to avoid merge hell.
- **Money-moving paths strictly additive.** `level_guard_daemon.py`, `api/trades.py` order-placement, `api/open_orders.py` order-placement, `api/suspend.py` close-flows — only logging, caches, LIMIT 1, dead-code deletion, or parity-test-protected helper extraction.
- **Re-triaged T2 queue when user waived sign-off.** Rather than executing all 135 T2s blind, split into T2a (canonical answer exists, tests verify, strictly additive) vs T2b (genuine design call). Executed only T2a (22 items).
- **Accepted tracker-row corrections where audit was wrong.** AUD-0186 (symlink, `.resolve()` follows symlinks — never buggy), AUD-0203/0204/0205 (enums are used in other files the auditor didn't check). Flipped to Resolved with WAI / false-positive notes.
- **Halted rather than force-fit.** 5 items marked T2a turned out to have semantic surprises during execution: AUD-0031 (legitimate standalone callers), 0177 (already had sufficient logging), 0260 (`_expand_env_vars` semantic mismatch), 0275 (QUICK vs NORMAL values differ), 0174-site-2 (part of AUD-0147 cluster). Sub-agents correctly left them as T2b.

- **AUD-0186 was a false positive.** `bin/levelguard_cli.py` was a SYMLINK, and `Path(__file__).resolve()` follows symlinks on Linux — so the claimed "path-resolution bug produces /app/syb/tradesuite instead of .../tradelens" never actually fired. Deletion was still right (duplication cleanup) but the original audit reasoning was wrong. I over-committed to the audit's narrative in my first tracker-flip commit; pushed a follow-up `232400a2` to correct it. **Lesson: verify auditor claims with `git show` before committing to their narrative.**
- **mdsync package still exists.** Memory says `lib/tradelens/mdsync/` was deleted in March 2026 — but `ls` showed it's still there (fetcher.py, runner.py, config.py, etc.). The memory was about a different cleanup. AUD-0285, AUD-0275, etc. target real files.
- **pytest baseline moved mid-session.** I measured 521 after Phase 1, sub-agent measured 549 before Group 2. The difference is probably that my measurement excluded phase3-research test files that fail to collect (ModuleNotFoundError on `tradelens.swing_research.phase3_features`). Not an issue for the run — every sub-agent's local pytest run stayed consistent within its own scope.
- **AUDIT_TRACKER.md concurrent-edit hazard.** Two actors (me + a sub-agent) trying to edit the tracker in parallel produced a "file has been modified since read" error. Solution: let sub-agents own tracker flips for their batches; parent only edits when no sub-agent is active.
- **AUD-0029 (bybit_client error convention)** — broadening `except` to `Exception` technically swallows `KeyboardInterrupt`/`SystemExit` that inherit from `BaseException` (not `Exception`), so those still propagate correctly. This is a subtlety worth flagging in future batches.
- **`_expand_env_vars` has TWO implementations.** `core/account_context.py` has a raising instance method; `api/discord_ingest.py` had a silent-default local copy. AUD-0260 wanted to consolidate, but the semantics differ. Correctly halted to T2b.
- **AUD-0275 tracker claim was wrong.** `QUICK_TIMEFRAME_CONFIG` and `TIMEFRAME_CONFIG` share keys but have DIFFERENT VALUES — not a strict subset. Lookbacks 30d vs 365d for `1h`. Not a derivation candidate.
