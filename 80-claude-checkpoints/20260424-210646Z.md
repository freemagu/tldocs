## Decisions made (and why)

- **Option 2 (trainer writes to vault), not option 1 (manual copy) or 3
  (dual-write).** Reasons: the pre-commit hook already expects new MDs to live
  under `docs/`; the MD is a generated report, not a research data artefact;
  dual-write would create drift risk for anyone hand-editing. User agreed.
- **Keep CSVs at `research/swing_levels/phase3/`.** Two reasons: the
  consolidation doc deliberately left CSVs out of the vault ("bulk data, not
  part of the Obsidian vault"), and my existing pipeline code writes
  `features.csv` + `importance/*.csv` there. Splitting MD output from data
  output in `_resolve_paths` keeps the convention consistent.
- **Explicit if/else in `_resolve_paths`** instead of the original terse
  inline-ternary. Adds a ~6 line docstring explaining why data and docs have
  separate bases. Small readability win for non-obvious convention.
- **Did NOT delete the old `research/swing_levels/phase3/phase3_results.md`.**
  The consolidation already did that via `git mv`. Verified absent before
  trainer run.

## Rejected approaches (and why)

- **Option 1: keep trainer writing to `research/`, manually copy to vault.**
  Rejected — guarantees staleness every time you run the trainer and forget
  to copy.
- **Option 3: dual-write (both locations).** Rejected — looks safe but is
  actually the worst: if anyone hand-edits one copy, the other silently
  diverges on the next run. Vault is supposed to be source of truth for MDs.
- **Keeping the inline-ternary form** `foo = a if sub else b` across three
  lines. Rejected — readable enough for two outputs but gets line-noisy with
  three; the if/else is clearer for the data/docs split.
- **Rebuilding the venv myself** when it broke. Rejected — environment
  repair is the user's call; I flagged + stood by.
- **Running the full 35-min feature extraction to re-regenerate features.csv
  and then the trainer.** Rejected — features.csv hasn't changed and is
  byte-valid; only the trainer's write path changed. One 22-sec trainer run
  was enough to prove the new path works.
