2. **Finding:** `lib/tradelens/core/account_context.py:108` reads `load_config().postgresql` — accessed via `load_config().postgresql` not `config.postgresql` so a static grep for the latter pattern misses it.
   **How we discovered it:** First BE gate had 115 failures + 81 errors, all cascading from `AttributeError: 'AppConfig' object has no attribute 'postgresql'` raised at `account_context.py:108`. Anything that built a `BybitClient` (which loads AccountContext) crashed.
   **Time cost:** ~10 minutes to triage + 5 minutes to do the broader grep `\.postgresql\b` and migrate the additional 4 callers I had missed.
   **Implication:** When deleting a config section, grep for ALL access patterns: `\.{section}\b` plus `cfg\['{section}'\]` plus `cfg\.get\('{section}'`. Static `config.postgresql` misses indirect access through `load_config().{section}`.
   **Where it's documented:** This checkpoint Surprise #2.

3. **Finding:** `lib/tradelens/locking/app_lock.py` has TWO YAML readers (`_load_db_config` line 276 and `_load_standalone_db_config` line 898) that read the YAML directly without going through AppConfig. Both used `cfg.get('postgresql', {})`.
   **How we discovered it:** Second BE gate had 11 failures including `test_amend_order_500_sanitizes_outer_leak` returning HTTP 409 instead of 500. The 409 came from `LockAcquireError` raised by the AppLock wrapper around `amend_order` — the AppLock was failing to load db_config because the YAML's `postgresql:` section was gone, falling through to env-var fallback which didn't have `PGUSER`/`PGPASSWORD` set in the test env.
   **Time cost:** ~10 minutes to triage.
   **Implication:** AppLock is a separate config-load path from the AppConfig schema. When deleting a config section, also grep for `cfg.get('{section}'` in any YAML reader. Note that AppLock's docstring even said "Try loading from YAML config first (postgresql section)" — that comment was load-bearing documentation.
   **Where it's documented:** This checkpoint Surprise #3 + the migrated `app_lock.py` now has AUD-0268 comments at both reader sites.
